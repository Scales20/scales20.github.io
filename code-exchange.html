Code Exchange

Context voor en rol binnen het project

Dit was het tweede project dat ik voor de propedeuse hoorde te doen. In dit project moest ik, samen met een teamgenoot, een website bouwen waarop mensen vragen over code kunnen stellen, beantwoorden, en vragen en antwoorden een rating kunnen geven. Hoewel het de bedoeling was dat ik en mijn teamgenoot gelijke delen zouden maken, heb ik het grootste deel van de eigenlijke functionaliteit gemaakt, inclusief de functionaliteit voor vragen en antwoorden en de ratings.

Omschrijving en afbeeldingen

De Code Exchange is bedoeld als een afgeleide van websites zoals StackExchange. Gebruikers kunnen zich hier registreren en inloggen (een account is nodig om vragen en antwoorden te posten); gebruikers kunnen ook hun wachtwoord resetten. Op de website kunnen gebruikers een profiel aanmaken, waar zij kunnen aangeven hoeveel ervaring zij hebben met coderen ("high", "medium", of "low") en met welke programmeertalen zij bekend zijn. Zij kunnen hier ook een profielfoto uploaden en hun account verwijderen. Gebruikers kunnen vragen stellen (en deze taggen met relevante programmeertalen en bewerken); bij de vraag wordt hun expertise getoond. Elke vraag kan worden beantwoord, en deze antwoorden kunnen altijd worden bewerkt. Elk antwoord kan ook een rating (op een schaal van 1 tot 5) krijgen, die ook kan worden bewerkt. Deze rating is bij het desbetreffende antwoord te zien. De gebruiker kan zowel alle vragen en antwoorden zien als alleen die vragen en antwoorden die hij zelf heeft gepost. Op de pagina waar alle vragen zijn te zien, is het ook mogelijk om alleen beantwoorde vragen te tonen. Hieronder is te zien hoe de website eruit ziet: De thuispagina. Een vraag met voorbeelden van antwoorden. Het scherm om een vraag te stellen. De profielpagina. De pagina waar een gebruiker een rating kan toevoegen.

Waarop ben ik trots?

Ik ben trots op dat ik een functionele website heb weten te maken, en vooral op de manier waarop ik de antwoorden bewerkbaar heb weten te maken. Dit was iets wat ik nog niet eerder had gedaan, en door de antwoorden bewerkbaar te maken, heb ik de website aanzienlijk bruikbaarder gemaakt. Ik ben verder blij met de oplossing die ik heb gebruikt om de code in antwoorden gescheiden te houden van de code in de pagina, door de "<" en ">" tekens te vervangen door "<" en ">". Uiteindelijk ben ik nog blij met het ontwerp van de ratingspagina.

Welke vaardigheden heb ik hier laten zien?

Ik heb de volgende vaardigheden getoond in dit project: * Verdere bekendheid met HTML. * Matige kennis van Typescript. * Samenwerking met een teamgenoot.
